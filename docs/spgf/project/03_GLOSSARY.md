# Glossary

This document defines project terms to keep business analysis, requirements, and implementation aligned.

## Terms

### Branch

The restaurant location managed by the system. Phase one targets one branch only.

### Core Inventory Item

An item selected for daily counting in the first phase. The final list is not yet defined.

### Daily Inventory Count

A recorded count of core inventory items for a specific day.

### Count Snapshot

The saved state of counted inventory items at a specific date/time, including the user who entered the count.

### Next-Day Matching

The process of comparing inventory count records across days to identify mismatches that require review. The exact matching rule is still open.

### Mismatch

A difference between count records or expected values that requires review.

### POS

The existing local/basic point-of-sale system used by the restaurant. It is not being replaced in phase one.

### SQL Server

The database engine used by the existing POS and installed on the cashier computer.

### Owner

The restaurant owner. Expected to have full access and management visibility.

### Manager

The operational manager. Expected to review inventory and operational data.

### Cashier

The user operating or related to the cashier/POS workflow.

### Relevant Employee

An employee who may participate in inventory counting or related operational tasks with limited permissions.

### Waste

Inventory loss or consumption not explained by normal sale/use. Future phase, not phase one.

### Delivery-App Reconciliation

Matching restaurant records with delivery app records. Future phase, not phase one.
